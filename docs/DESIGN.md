# 디자인 규정 (포트폴리오 기준)

## 1. 디자인 철학 (포트폴리오 스토리)

이 앱은 **"작은 데스크톱 셸"**을 모티브로 합니다.

- **패널**: 정보를 정리합니다.
- **절제된 카드**: 데이터를 보여줍니다.
- **그래프 화면**: 관계를 시각화합니다.
- **모달**: 작업을 수행합니다.
- **토큰 기반 테마**: 일관성을 유지합니다.

**시각적 특징**:
- **아날로그 모드 (Light)**: `Aptos` (영문), `Noto Serif KR` (한글). 종이 인쇄물 느낌.
- **디지털 모드 (Dark)**: `Segoe UI` (영문), `Noto Sans KR Light` (한글). 현대적이고 명확한 디지털 서체.

**포트폴리오 강조점**:
- **편집 디자인식 레이아웃**: 여백을 활용한 시각적 휴식.
- **타이포그래피 중심**: 글자 크기와 굵기로 위계 표현.
- **절제된 효과**: 그림자와 그라디언트 최소화.
- **그래프 상호작용**: 기능적으로 분명하되 시각적으로 과하지 않음.

## 2. 시각 규정 (에이전트 준수 사항)

### 2.1. 색상 (Color)

- **규정**: 한 화면에 최대 3가지 색상만 사용합니다. 배경색(`--page`), 텍스트색(`--text`), 액센트색(`--accent`).
- **규정**: 다른 색상은 `variables.css`에서 정의된 토큰만 사용합니다. 임의의 색상 코드는 금지합니다.
- **규정**: 빨강/코랄 계열(`--coral`)은 위험, 삭제, 실제 병목 경고에만 사용합니다.

### 2.2. 타이포그래피 (Typography)

- **규정**: 아래 크기 외의 `font-size`는 사용하지 않습니다.
    - **32px**: 상세 헤더 프로젝트 타이틀 (`.detail-header h2`)
    - **28px**: 핵심 지표 숫자 (`.detail-progress strong`)
    - **22px**: 앱 타이틀 (`.brand h1`)
    - **16px**: 패널 섹션 헤더 (`.section-heading h2`)
    - **14px**: 일반 텍스트, 버튼 (`.project-item-top strong`)
    - **12px**: 보조 라벨, 필터 탭 (`.tabs button`)
    - **11px**: 메타 정보 칩 (`.project-item-meta`)
    - **10px**: 그래프 하단 라벨 (`.project-mini-graph small`)

### 2.3. 간격 (Spacing)

- **규정**: margin/padding은 `8px`, `16px`, `24px`, `32px`, `48px`만 사용합니다. 다른 값은 사용하지 않습니다.
- **규정**:
    - 카드 내부 여백: 16px
    - 섹션 간 간격: 24px
    - 패널 간 간격: 32px

### 2.4. 효과 (Effects)

- **규정**: 임의의 그라디언트나 과한 그림자를 피합니다.
- **규정**: 장식 효과보다 테두리, 간격, 글자 위계를 우선합니다.
- **규정**: `variables.css`의 토큰(`--shadow`, `--focus-ring`)만 사용합니다.

## 3. 금지 사항

- 흔한 관리자 대시보드식 카드 남발 금지.
- 마케팅 페이지처럼 큰 히어로 레이아웃 금지.
- 장식용 그라디언트나 배경 효과 금지.
- 과한 그림자와 떠 있는 패널 금지.
- 구현 설명을 그대로 노출하는 UI 문구 금지.
- 작은 버튼/카드/패널에서 넘치는 텍스트 금지.
- 같은 개념의 진행도를 여러 방식으로 보여주면서 원천을 설명하지 않는 것 금지.

## 4. 작업 규칙

- 스타일을 바꾸기 전 `docs/FRONTEND.md`와 기존 CSS를 읽습니다.
- 변경 범위는 수정하는 화면에 한정합니다.
- analog/digital 테마 동작을 유지합니다.
- 좁은 창에서 텍스트가 넘치지 않는지 확인합니다.
- 사용자가 요청하지 않으면 그래프나 앱 셸을 리디자인하지 않습니다.
- 새 상태를 추가하면 저장 상태인지, 파생 상태인지, 화면 전용 상태인지 문서화합니다.
