# 디자인 원칙

## 방향

이 앱은 조용하고, 우아하고, 구조적이며, 편집 디자인처럼 여백이 살아 있는 도구여야 합니다. 일반적인 빽빽한 SaaS 관리자 대시보드처럼 보이지 않아야 합니다.

현재 디자인은 작은 데스크톱 셸, 패널, 절제된 카드, 그래프 화면, 모달, 토큰 기반 테마로 구성되어 있습니다. 앞으로의 UI 작업은 이 방향을 다듬는 방식이어야 하며, 명시 요청 없이 갈아엎지 않습니다.

## 시각 톤

- 조용하고 집중된 느낌.
- 구조적이지만 딱딱하지 않은 화면.
- 가능한 곳에서는 편집 디자인식 여백.
- 타이포그래피, 간격, 대비로 만드는 명확한 위계.
- 절제된 효과.
- 그래프 상호작용은 기능적으로 분명하되 시각적으로 과하지 않게.

## 타이포그래피

### 실제 사용 중인 폰트

이 앱은 화면 모드(아날로그/디지털)에 따라 다른 영문 및 한글 폰트를 대응시켜 디자인의 심미성과 가독성을 통제합니다.

- **아날로그 모드 (Analog / Light 테마)**
  - **영문/숫자 (Latin)**: `"Aptos"` (구조적이고 조용한 샌스세리프 느낌)
  - **한글 (Korean)**: `"Noto Serif KR"` (종이 인쇄물 느낌의 부드러운 명조체)
- **디지털 모드 (Digital / Dark 테마)**
  - **영문/숫자 (Latin)**: `"Segoe UI"` (현대적이고 명확한 디지털 서체)
  - **한글 (Korean)**: `"Noto Sans KR"` (가독성 높은 고딕체)

### 타이포그래피 위계 (Hierarchy)

- **32px**: 상세 헤더의 핵심 프로젝트 타이틀 (`.detail-header h2`, `line-height: 1.12`)
- **28px**: 핵심 진행률/완성도 지표 숫자 (`.detail-progress strong`, `font-variant-numeric: tabular-nums`)
- **22px**: 최상단 앱 타이틀 (`.brand h1`)
- **16px**: 목록 내 완성도/진행도 수치 (`.completion-badge b`), 패널 및 섹션별 대표 헤더 (`.section-heading h2`, `.task-section-title h3`)
- **15px**: 최상단 앱 서브 타이틀 (`.brand p`)
- **14px**: 프로젝트 아이템 제목 (`.project-item-top strong`), 새 프로젝트 추가 버튼 (`.bottom-command`)
- **12px**: 필터 탭 메뉴 버튼 (`.tabs button`), 프로젝트 상세 속성 보조 라벨 (`.detail-progress span`)
- **11px**: 프로젝트 목록 메타 정보 칩 (`.project-item-meta`), 가중치 비율 숫자 (`.breakdown-number`)
- **10px**: 그래프 하단 라벨 및 배지 보조 소제목 (`.project-mini-graph small`, `.completion-badge small`)

### 디자인 적용 규칙

- 컨트롤과 메타 정보에는 작은 보조 라벨(10px ~ 11px)을 사용합니다.
- 패널 안 제목은 중간 굵기와 크기(14px ~ 16px)를 사용합니다.
- 큰 글자(28px ~ 32px)는 상세 헤더와 핵심 프로젝트 지표에만 사용합니다.
- 진행도, 카운터, 지표 숫자는 가독성을 위해 가능하면 고정폭 숫자인 `tabular-nums` (`font-variant-numeric: tabular-nums`)를 명시합니다.
- 기존 패턴이 아니라면 글자 간격을 과하게 조정하지 않습니다.

## 간격

- 촘촘한 표보다 의도적인 섹션 간격을 우선합니다.
- 패널은 스캔하기 쉬운 정도의 밀도를 유지합니다.
- 카드 안에 다시 카드를 넣지 않습니다.
- 그래프 컨트롤과 오버레이가 중요한 그래프 내용을 가리지 않게 합니다.
- 모달 폼은 작고 예측 가능하게 유지합니다.

## 컴포넌트 재사용

새 컴포넌트를 만들기 전에 기존 패턴을 먼저 확인합니다.

- 프로젝트 목록 카드
- 상세 헤더와 지표 카드
- 작업 카드
- 아카이브 리소스 행
- 그래프 프로젝트 카드
- 그래프 자유 노드
- 모달 폼 패턴
- 세그먼트/탭 컨트롤
- `variables.css`의 토큰

새 UI 위치:

- 일반 상세/목록 마크업: `ui-components.js`
- 그래프 마크업: `graph-components.js`
- 모달 동작: `app-modals.js`
- 이벤트 연결: `app-graph-events.js`

## 색과 효과

- 기존 디자인 토큰을 사용합니다.
- 임의의 그라디언트를 추가하지 않습니다.
- 과한 그림자를 피합니다.
- 한 가지 색상군만 지배하는 팔레트를 피합니다.
- 빨강/코랄 계열은 위험, 삭제, 실제 병목 경고에만 사용합니다.
- 장식 효과보다 테두리, 간격, 글자 위계를 우선합니다.

## 아이콘

현재 앱은 inline SVG와 텍스트 기호를 함께 씁니다. 앞으로 아이콘을 추가할 때는 장난스러운 아이콘을 피하고, 기능을 설명하는 조용한 아이콘만 사용합니다.

## 피해야 할 UI

- 흔한 관리자 대시보드식 카드 남발.
- 마케팅 페이지처럼 큰 히어로 레이아웃.
- 장식용 그라디언트나 배경 효과.
- 과한 그림자와 떠 있는 패널.
- 구현 설명을 그대로 노출하는 UI 문구.
- 작은 버튼/카드/패널에서 넘치는 텍스트.
- 같은 개념의 진행도를 여러 방식으로 보여주면서 원천을 설명하지 않는 것.

## 향후 Codex UI 작업 규칙

- 스타일을 바꾸기 전 `docs/FRONTEND.md`와 기존 CSS를 읽습니다.
- 변경 범위는 수정하는 화면에 한정합니다.
- analog/digital 테마 동작을 유지합니다.
- 좁은 창에서 텍스트가 넘치지 않는지 확인합니다.
- 사용자가 요청하지 않으면 그래프나 앱 셸을 리디자인하지 않습니다.
- 새 상태를 추가하면 저장 상태인지, 파생 상태인지, 화면 전용 상태인지 문서화합니다.
