# Bottleneck Recommendations Design

## Goal

기존 병목 카드에 “다음에 무엇을 하면 좋은지”를 짧게 붙입니다. 새 화면이나 저장 상태를 만들지 않고, 기존 병목 계산과 롤업 설명 데이터를 파생해서 사용합니다.

## Chosen Approach

1안으로 진행합니다. 기존 병목 카드 안에서 각 병목 항목에 추천 문장을 추가합니다.

## Data Flow

- `calculator.js`가 `getBottleneckDetails()`와 `getRollupExplanation()`를 조합해 `getBottleneckRecommendations(projectId)`를 반환합니다.
- 추천은 저장하지 않습니다.
- `ui-components.js`는 추천 객체를 받아 병목 카드에 “추천” 문장을 렌더링합니다.

## Recommendation Rules

- 할 일 병목: 해당 할 일을 집중 대상으로 삼도록 권합니다.
- 하위 프로젝트 병목: 해당 프로젝트를 열어 낮은 할 일을 정리하도록 권합니다.
- 외부 프로젝트 병목: 외부 반영 원천을 추적하거나 할 일을 추가하도록 권합니다.
- 수식 병목: 수식 원천을 추적해서 입력값을 확인하도록 권합니다.
- 완성도와 진행도 병목을 모두 볼 수 있게 하되, 영향도가 큰 항목이 먼저 오도록 정렬합니다.

## Non-Goals

- 자동으로 작업을 생성하지 않습니다.
- 상태나 진행도 값을 새로 저장하지 않습니다.
- 병목 점수 엔진이나 별도 추천 대시보드는 만들지 않습니다.

## Acceptance Criteria

- 병목 카드에 추천 문장이 표시됩니다.
- 추천은 기존 추적/할 일 추가/집중 버튼과 함께 작동합니다.
- 진행도 전용 병목도 추천 대상에 포함됩니다.
- 전체 테스트가 통과합니다.
