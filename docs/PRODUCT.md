# 제품 범위 규정

## 1. 제품 정의

- **규정**: 이 앱은 프로젝트, 하위 프로젝트, 작업, 진행 상태, 마감, 관련 리소스를 관리하는 작고 상시 표시 가능한 데스크톱 프로젝트 관리 앱입니다.
- **규정**: Electron으로 실행되며 데이터는 브라우저 `localStorage`에 로컬 저장됩니다.
- **규정**: UI와 패키지 메타데이터에는 한국어 이름이 들어 있습니다. 한국어 문구를 바꿀 때는 UTF-8로 저장하고, 사용자에게 보이는 문구는 실제 렌더링 화면에서 확인해야 합니다.

## 2. 대상 사용자

- **규정**: 개인 또는 소규모 창작/업무 사용자가 프로젝트의 구조와 진행 상태를 빠르게 파악하기 위한 작업실형 도구입니다.
- **규정**: 팀용 SaaS보다는 개인의 프로젝트 조종석에 가깝습니다.
- **가정**: 이 앱은 개인 프로젝트 추적과 창작 작업 관리에 초점이 있으며, 대규모 협업 관리 도구를 목표로 하지 않습니다.

## 3. 핵심 목표

- **규정**: 사용자가 어떤 일이 있는지, 어떤 프로젝트가 어떤 구조 아래 있는지, 무엇이 지연되고 있는지, 작업과 하위 프로젝트가 상위 진행도에 어떻게 반영되는지 신뢰할 수 있게 보여주는 것이 핵심입니다.
- **규정**: 특히 프로젝트 계층과 완성도/진행도 반영이 서로 어긋나지 않아야 합니다.

## 4. 해결하는 문제

- **규정**:
    - 프로젝트와 하위 프로젝트를 계층 구조로 관리합니다.
    - 프로젝트 아래 작업을 관리합니다.
    - 완성도와 진행도를 별도 값으로 보여줍니다.
    - 낮은 진행도의 하위 항목이나 외부 영향이 상위 프로젝트에 주는 병목을 보여줍니다.
    - 파일, 폴더, URL 같은 리소스를 프로젝트나 작업에 연결합니다.
    - 그래프 화면에서 계층, 관계, 수식, 메모, 아카이브 노드, 작업 배치를 시각적으로 다룹니다.
    - 작은 집중 위젯으로 현재 작업을 따로 볼 수 있습니다.
    - JSON 가져오기/내보내기로 로컬 데이터를 백업합니다.

## 5. 목표가 아닌 것

- **규정**: 현재 코드 기준으로 이 앱은 다중 사용자 SaaS, 캘린더, 채팅/협업 도구, 이슈 트래커, 문서 편집기를 목표로 하지 않습니다.
- **규정**: 사용자가 명시하지 않는 한 일반적인 관리자 대시보드 방향으로 확장하지 않습니다.

## 6. 주요 사용자 흐름

- **규정**:
    - 프로젝트를 만들고, 필요하면 상위 프로젝트, 상태, 마감, 메모를 지정합니다.
    - 프로젝트에 작업을 추가하고 완성도, 진행도, 기여 방식을 설정합니다.
    - 작업 슬라이더를 조정하면 프로젝트 롤업 표시가 갱신됩니다.
    - 프로젝트의 상위 항목, 마감, 메모, 외부 영향 링크를 수정합니다.
    - 상세 화면에서 프로젝트 상태, 하위 프로젝트, 롤업 막대, 병목 알림, 작업, 연결된 아카이브 리소스를 확인합니다.
    - 그래프 화면에서 계층, 수식 노드, 메모 노드, 아카이브 노드, 작업 위치, 영향 링크를 다룹니다.
    - 아카이브 화면에서 리소스를 추가하고 프로젝트/작업에 연결하거나 분리합니다.
    - JSON으로 데이터를 내보내거나 가져옵니다.
    - 집중 위젯에서 현재 작업을 작게 관리합니다.

## 7. 불확실한 점

- **규정**:
    - 코드에는 별도 `Folder` 엔티티가 없습니다. 현재는 최상위 프로젝트나 중첩 프로젝트가 폴더처럼 쓰이는 것으로 보입니다.
    - 프로젝트 `status`는 저장된 텍스트이며, 작업 상태나 진행도에서 자동 파생되는지는 확인되지 않습니다.
    - 완성도와 진행도는 저장값과 파생 롤업값이 모두 존재합니다.
    - 한국어 문구는 UTF-8 기준으로 관리합니다. 인코딩 회귀가 의심되면 `test_encoding_integrity.mjs`를 먼저 실행합니다.
